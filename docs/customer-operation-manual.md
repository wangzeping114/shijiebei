# 2026 世界杯模拟竞猜系统操作文档

本文档基于当前项目代码整理，面向门店客户和后台运营人员，说明用户投注、后台赔率配置、开奖结算及赔付处理流程。

## 1. 系统概览

本系统为线下门店内部模拟竞猜系统，不涉及线上支付或真实资金托管。系统采用前后端分离结构：

- 前端：Vue 3 + Vite + Element Plus，提供用户投注端和管理后台。
- 后端：Node.js + Express + PostgreSQL，提供账号、赛事、赔率、投注、结算和报表接口。
- 默认访问路径：用户端为 `/`，管理后台为 `/admin`。

主要角色：

- 普通用户：注册/登录后查看赛事、选择投注项、提交投注、查看投注记录和中奖状态。
- 管理员：维护赛事、配置赔率、关闭接单、录入比分、自动结算、查看统计报表和投注上报文案。

默认管理员账号：

- 账号：`admin`
- 密码：`admin123`

生产环境交付时建议修改默认管理员密码，并确保后端环境变量中的 `JWT_SECRET` 使用强随机字符串。

## 2. 用户端投注操作流程

### 2.1 注册与登录

1. 打开系统登录页。
2. 新用户点击注册，填写用户名、昵称、密码。
3. 用户名长度要求为 3-20 个字符，密码不少于 6 位。
4. 注册成功后系统会自动登录；已有用户直接输入用户名和密码登录。

登录成功后，系统会保存登录状态。若登录过期或认证失败，系统会自动跳转回登录页。

### 2.2 查看赛事列表

用户登录后进入赛事列表页，可查看全部赛事。赛事状态包括：

| 状态 | 含义 | 用户是否可投注 |
| --- | --- | --- |
| 即将开赛 `upcoming` | 比赛尚未开始 | 可投注 |
| 进行中 `ongoing` | 比赛已开始且仍在系统开放窗口内 | 可投注 |
| 已关闭 `closed` | 管理员关闭接单，或系统自动关闭 | 不可投注 |
| 已结束 `finished` | 已录入比分并完成结算 | 不可投注 |

系统会根据比赛时间自动刷新部分状态：比赛开始后约 2 小时内标记为进行中，超过约 2 小时后自动关闭。

### 2.3 进入投注详情

1. 在赛事列表中选择一场赛事，进入赛事详情页。
2. 页面顶部展示主队、客队、比赛时间、轮次、场地和赛事状态。
3. 下方展示可投注内容，主要包括：
   - 比分投注
   - 市场盘口，如独赢、让球、大小球、双方是否进球、角球盘口等

赔率为 `0` 的选项表示未开盘，用户不能投注该选项。

### 2.4 比分投注

1. 在“比分投注”区域查看比分选项，如 `1 : 0`、`2 : 1`。
2. 在目标比分右侧填写投注金额。
3. 系统自动显示该投注项预计赔付：

```text
预计赔付 = 投注金额 x 当前赔率
```

4. 一个订单可同时选择多个比分，每个比分独立填写金额。
5. 页面下方会汇总已选投注项数量和总投注金额。

投注前，用户可在已选列表中修改或删除某个比分投注项。

### 2.5 市场盘口投注

若管理员已为赛事配置市场盘口，用户可在“市场盘口”区域投注。当前系统支持的盘口包括：

- 独赢盘口：主队独赢、平局、客队独赢。
- 上半场独赢盘口：上半场主队独赢、上半场平局、上半场客队独赢。
- 全场让球盘口。
- 上半场让球盘口。
- 全场比分大小盘口。
- 上半场比分大小盘口。
- 双方是否进球。
- 全场角球盘口。

操作方式与比分投注一致：选择盘口选项，填写金额，确认预计赔付。

### 2.6 提交投注并打印投注单

1. 确认已选投注项和总金额无误。
2. 点击“确认投注”。
3. 系统生成订单编号，并弹出格式化投注单。
4. 投注单包含赛事、玩法、开赛时间、投注项、赔率、投注金额、订单总额、订单编号和投注时间。
5. 可点击“打印投注单”，线下门店可将该单据作为兑奖凭证。

系统保存订单时会重新读取当前赔率，最终以提交瞬间后台数据库中的赔率为准。

### 2.7 查看和修改我的投注

用户可进入“我的投注记录”查看历史订单。订单状态包括：

| 状态 | 含义 |
| --- | --- |
| 待开奖 `pending` | 尚未结算 |
| 已中奖 `won` | 至少一个投注项中奖 |
| 未中奖 `lost` | 没有投注项中奖 |
| 已结算 `settled` | 预留状态，当前主要使用已中奖/未中奖 |

在满足以下全部条件时，用户可以编辑或删除投注项：

- 订单状态仍为待开奖。
- 赛事未开始。
- 赛事未关闭接单。
- 赛事未结算。

可修改内容：

- 比分投注项可修改为其他已开盘比分，并修改金额。
- 投注项可删除。
- 若订单中所有投注项都被删除，系统会删除该空订单。

市场盘口投注项当前在用户记录页仅支持删除，不支持改为其他盘口。

## 3. 后台赛事与赔率配置流程

### 3.1 进入管理后台

1. 使用管理员账号登录。
2. 登录后可通过页面入口进入 `/admin`。
3. 后台菜单包括：
   - 赛事管理
   - 比分录入
   - 投注上报
   - 统计报表

### 3.2 赛事管理

在“赛事管理”中，管理员可执行以下操作：

- 新增赛事：填写主队、客队、开赛时间。
- 编辑赛事：修改主队、客队、开赛时间和赛事状态。
- 删除赛事：删除赛事会连带删除关联赔率和投注数据，操作前需谨慎确认。
- 关闭接单：未开赛且未结束的赛事可手动关闭接单。
- 恢复赛事：已关闭或已结束赛事可恢复为“即将开赛”，但已录入比分和投注结算状态不会自动清除。

限制规则：

- 已结束赛事不可编辑、不可删除、不可重新配置赔率。
- 已开赛赛事不可进入赔率配置。
- 关闭接单后用户不能继续投注或修改订单。

### 3.3 比分赔率配置

进入方式：

1. 在“赛事管理”找到目标赛事。
2. 点击“赔率配置”。
3. 进入该赛事的赔率配置页面。

比分赔率支持以下操作：

- 一键生成默认赔率：系统生成 20 种常见比分，默认赔率均为 `0`。
- 手动新增比分：填写主队比分、客队比分、赔率。
- 修改赔率：直接在表格中调整赔率数值。
- 保存全部修改：批量保存当前页面所有比分赔率。
- 删除单条比分赔率。
- 清空全部比分赔率。
- 下载 Excel 模板。
- 导入 Excel 赔率。

默认比分包括：

```text
0:0, 1:0, 0:1, 1:1, 2:0, 0:2, 2:1, 1:2, 2:2,
3:0, 0:3, 3:1, 1:3, 3:2, 2:3, 4:0, 0:4, 4:1, 1:4, 3:3
```

配置规则：

- 赔率不能小于 0。
- 赔率为 0 表示未开盘，前端显示但不可投注。
- 同一赛事同一比分只能保留一条赔率，重复导入或新增会更新已有记录。

### 3.4 市场赔率配置

同一赔率配置页面还包含“市场赔率配置”。可配置以下盘口：

- 独赢盘口。
- 上半场独赢盘口。
- 全场让球盘口。
- 上半场让球盘口。
- 全场比分大小盘口。
- 上半场比分大小盘口。
- 双方是否进球。
- 全场角球盘口。

操作步骤：

1. 点击“生成默认市场赔率”。
2. 在弹窗中填写或确认盘口参数：
   - 全场让球线 1。
   - 全场让球线 2。
   - 上半场让球线。
   - 全场大小球线 1。
   - 全场大小球线 2。
   - 上半场大小球线。
3. 点击生成后，系统按主客队名称生成默认盘口，赔率默认为 `0`。
4. 在表格中填写各选项赔率。
5. 点击“保存市场赔率”。

市场赔率还支持：

- 下载市场赔率模板。
- 导入市场赔率 Excel。
- 删除单条市场赔率。
- 清空全部市场赔率。
- 新增角球线，如“全场角球大于 9.5”。

注意事项：

- 让球和大小球盘口重新生成时，系统会先删除该赛事旧的让球/大小球行，再按新参数生成。
- 独赢、双方是否进球、角球行不会因调整让球/大小球参数而自动删除。
- Excel 导入市场赔率时，建议不要修改“选项代码”列；系统根据选项代码匹配更新。

## 4. 投注上报流程

后台“投注上报”用于门店核对订单，并复制汇总文案进行外部上报。

操作步骤：

1. 进入后台“投注上报”。
2. 选择订单范围：
   - 仅待开奖订单。
   - 全部订单。
3. 可按用户、订单号、赛事、比分或盘口关键字搜索。
4. 在上方表格核对明细，包括赛事、用户、订单号、投注项、赔率、金额、状态和投注时间。
5. 在下方“可复制上报文案”区域查看按赛事汇总后的文案。
6. 点击“复制上报文案”，粘贴到指定上报渠道。

上报文案按赛事聚合，同一赛事下相同投注项和相同赔率会合并金额。

## 5. 开奖与自动结算流程

### 5.1 录入上半场比分

若需要结算上半场盘口，可先录入上半场比分：

1. 进入后台“比分录入”。
2. 找到目标赛事。
3. 在“上半场”区域输入主队和客队上半场比分。
4. 点击“录入上半场”。
5. 系统结算上半场相关市场投注，但不改变赛事状态。

上半场结算范围：

- 上半场独赢盘口。
- 上半场让球盘口。
- 上半场比分大小盘口。

### 5.2 录入全场比分并结算

全场比赛结束后执行：

1. 进入后台“比分录入”。
2. 找到目标赛事。
3. 在“全场”区域输入主队和客队最终比分。
4. 如配置了角球盘口，可填写主队角球数和客队角球数；不填写则不结算角球盘口。
5. 点击“确认录入 & 自动结算”。
6. 系统弹出确认框，确认后自动完成结算。

该操作会：

- 将赛事状态更新为“已结束”。
- 保存最终比分和角球数。
- 标记命中最终比分的比分投注项为中奖。
- 结算全场市场盘口。
- 如果已录入上半场比分，也会一并确保上半场市场盘口完成结算。
- 将未中奖订单状态置为 `lost`。
- 将包含至少一个中奖项的订单状态置为 `won`。

全场比分录入为不可撤销的关键操作，录入前应与赛果核对无误。

## 6. 中奖判定规则

### 6.1 比分投注

若用户投注的比分与后台录入的全场比分完全一致，则该投注项中奖。

示例：

```text
用户投注：主队 2 : 1 客队
后台录入：主队 2 : 1 客队
结果：中奖
```

### 6.2 独赢盘口

- 主队独赢：主队比分大于客队比分。
- 平局：主队比分等于客队比分。
- 客队独赢：客队比分大于主队比分。

上半场独赢使用上半场比分判定；全场独赢使用全场比分判定。

### 6.3 大小球盘口

- 大于 N：主队比分 + 客队比分 > N。
- 小于 N：主队比分 + 客队比分 < N。

当前系统没有走水/退回本金逻辑；若总进球数等于盘口线，则不标记中奖。

### 6.4 让球盘口

系统代码中的判定方式为：

```text
主队让球项中奖：主队比分 + 让球值 > 客队比分
客队让球项中奖：客队比分 + 让球值 > 主队比分
```

上半场让球使用上半场比分判定；全场让球使用全场比分判定。

### 6.5 双方是否进球

- 双方均进球（是）：主队比分 > 0 且客队比分 > 0。
- 双方未均进球（否）：不满足双方均进球条件。

### 6.6 全场角球盘口

系统支持“全场角球大于 N”：

```text
主队角球数 + 客队角球数 > N
```

只有管理员在录入全场比分时同时填写了双方角球数，角球盘口才会参与结算。

## 7. 赔付处理流程

### 7.1 赔付金额计算

系统按投注项计算中奖赔付：

```text
单项赔付金额 = 投注金额 x 投注时锁定赔率
```

订单可包含多个投注项，只要其中任意投注项中奖，订单状态即为“已中奖”。订单中奖总额为该订单所有中奖项赔付金额之和。

### 7.2 用户端查看赔付

用户进入“我的投注记录”后：

- 未开奖时显示“待开奖”。
- 已结算且未中奖时显示“未中奖”。
- 中奖投注项显示“中奖 +赔付金额”。
- 订单底部显示总投注金额和中奖赔付金额。

### 7.3 后台统计报表

管理员可进入“统计报表”查看已结束赛事的赔付情况。

报表包含：

- 总投注笔数。
- 总投注金额。
- 中奖人数。
- 总赔付金额。
- 中奖用户名单。
- 所有用户投注明细。

管理员可筛选已结束赛事，查看每场比赛的中奖用户和明细，并可打印报表。

### 7.4 门店兑奖建议

建议门店按以下步骤处理线下兑奖：

1. 用户出示打印投注单或提供订单编号。
2. 管理员在后台统计报表或用户投注明细中核对订单。
3. 核对内容包括：订单编号、用户名/昵称、赛事、投注项、投注金额、赔率、中奖状态和赔付金额。
4. 核对无误后，按系统显示的中奖赔付金额进行线下兑奖。
5. 若存在争议，以后台数据库订单记录和已录入赛果为准。

当前系统只计算中奖赔付金额，没有独立的“已兑付/未兑付”状态流转。若客户需要记录实际付款状态，建议后续增加兑付登记功能。

## 8. 常见运营注意事项

- 开盘前务必检查赔率是否大于 0；赔率为 0 的投注项不会开放投注。
- 比赛临近开赛前，如不再接受投注，应在“赛事管理”中手动关闭接单。
- 已开赛或已关闭的赛事，用户不能修改或删除投注项。
- 录入全场比分会触发自动结算，操作前必须核对比分。
- 若需要角球盘口结算，录入全场比分时必须同步填写角球数。
- 恢复已结束赛事不会清除原比分和原订单结算状态，谨慎使用。
- 删除赛事会删除关联赔率和投注数据，不建议在已有投注后执行。
- 市场赔率 Excel 导入时，请保留选项代码列，避免导入后无法按预期结算。
